# Playbook: Long content generation

Паттерн "генерация большого объёма контента (20+ единиц или статья 5000+ слов) без монотонности и качественной деградации".

## Когда применять

- Серия постов/статей по одной теме (TG-канал, блог, рассылка)
- Лонгрид >5000 слов
- Multi-chapter material (книги, курсы, документация)

## Принципы

1. **Структура → секции → драфты → финал.** Никогда не пиши единым потоком.
2. **Каждый кусок отдельным файлом.** `posts/01-intro.md`, `posts/02-history.md`, etc. Это спасает контекст.
3. **Variation policy** — не позволяй pattern lock-in (одинаковая структура постов, повторяющиеся обороты, monolithic Tone).
4. **Reference-driven** — каждое утверждение должно опираться на research-материалы. Цитата + URL.
5. **Никогда не сокращай при склейке.** Manus принцип: финальная длина >= сумма драфтов. Сокращение происходит на этапе draft, не compilation.

## Workflow для серии (например 30 постов TG-канала)

1. **Стратегия** (отдельный playbook): pillars, ToV, ЦА, частота
2. **План постов** в `posts/_plan.md`:
   ```
   # План — 30 постов

   | # | Pillar | Тип | Hook | Заголовок |
   |---|---|---|---|---|
   | 1 | Pillar A | history | "Знал ли ты что..." | ... |
   | 2 | Pillar B | how-to  | "5 способов..." | ... |
   | ... |
   ```
3. **Идеи по постам** — по одному файлу `posts/01-<slug>.md`:
   ```markdown
   # Заголовок

   ## Цель / hook
   ...

   ## Структура
   - intro
   - body (3 части)
   - заключение
   - CTA

   ## Ключевые факты (из research/)
   - ...

   ## Variation notes
   - не используй "вот так" (использовал в посте 02)
   - этот пост короче (предыдущий был длинный)
   ```
4. **Drafting** — пиши пост, после написания **обнови `posts/_variation_log.md`** с используемыми оборотами/структурами.
5. **Quality check** перед финалом: прочитай 3 последних поста подряд — нет ли patterning?
6. **Compilation** — если нужен единый документ, append'ом склей файлы:
   ```python
   for f in sorted(Path("posts").glob("*.md")):
       file_write("final.md", f.read_text(), append=True, leading_newline=True)
   ```

## Workflow для лонгрида

1. **Outline** в `outline.md` — все секции с 1-2 строчным описанием
2. **Section drafts** — каждую секцию отдельным файлом `sections/<slug>.md`
3. **Per-section research** в `research/<topic>.md` (см. research playbook)
4. **Compilation** — append'ом в `final.md`. **Никаких суммаризаций при склейке.**

## Variation policy (детально)

Для серии >10 единиц веди `_variation_log.md`:

```markdown
# Variation log

## Открытия
- "Знал ли ты что..." — пост 1, 7
- "5 фактов о..." — пост 3
- "Вопрос..." — пост 5

## Структуры
- факт + объяснение + CTA — посты 1, 4
- история + мораль — посты 2, 6

## Tone
- спокойный — посты 1-3
- провокационный — пост 4
- юмор — пост 7
```

Перед написанием нового поста проверяй log → выбирай неиспользованную или мало-используемую комбинацию.

## Антипаттерны

- ❌ Писать сразу финальный документ единым потоком
- ❌ Не вести variation log → серия становится монотонной
- ❌ Сокращать при склейке (финальный текст должен быть длиннее суммы драфтов)
- ❌ Утверждения без reference-материала
- ❌ Использовать одну и ту же структуру для всех 30 постов
